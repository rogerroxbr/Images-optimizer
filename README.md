# Academic Image Optimizer

A smart, lightweight Python CLI tool specifically designed to optimize figures, charts, and diagrams for academic papers. 

Many journals and pre-print servers (like arXiv) impose strict file size limits on PDF submissions. High-resolution figures generated by matplotlib, seaborn, or exported from design tools often have unnecessarily large dimensions, transparent backgrounds (which can cause PDF rendering issues), or unoptimized color channels. 

This script solves these issues by intelligently downscaling extremely large dimensions using the high-quality Lanczos resampling filter, flattening alpha transparency channels into solid white backgrounds (to avoid black-background artifacts in LaTeX/PDF conversion), and applying lossless/lossy size reduction suitable for print and web reading.

## Features
- **Smart Filtering:** Only optimizes images that breach a certain dimension threshold or file size, skipping already-optimized assets.
- **Academic Standard:** Strips alpha channels (transparency) and replaces them with a white background, preventing inverted-color bugs when compiling LaTeX PDFs.
- **High-Quality Resampling:** Utilizes `LANCZOS` downsampling to preserve fine lines, text, and data points in plots.
- **Batch Processing:** Safely processes entire directories of `.png`, `.jpg`, and `.jpeg` images.

## Requirements

The only dependency is the widely-used Python Imaging Library (`Pillow`).

```bash
pip install Pillow
```

## Usage

You can invoke the script from the command line, providing an input directory.

```bash
python optimize_images.py -i <path_to_input_directory>
```

### Options

| Argument | Short | Default | Description |
| :--- | :--- | :--- | :--- |
| `--input` | `-i` | **Required** | The source directory containing your images. |
| `--output` | `-o` | `<input_dir>` | The destination directory for optimized images. If omitted, the script will overwrite the original files. |
| `--max-dim` | | `1600` | The maximum allowable width or height (in pixels). If an image exceeds this, it is proportionally downscaled. |
| `--max-size-kb` | | `200` | Minimum file size (in KB) to trigger optimization. Images smaller than this are skipped to save processing time. |

### Examples

**1. Basic Optimization (Overwrite Originals)**
Perfect for a quick pass over your LaTeX `figures/` folder before submitting.
```bash
python optimize_images.py -i ./figures
```

**2. Save to a Separate Folder**
If you want to preserve your ultra-high-res originals and export the web/PDF versions to a `build/` folder.
```bash
python optimize_images.py -i ./raw_figures -o ./optimized_figures
```

**3. Aggressive Size Reduction**
For extremely strict submission portals, force a maximum dimension of 1000px and compress anything over 100 KB.
```bash
python optimize_images.py -i ./figures --max-dim 1000 --max-size-kb 100
```

## How It Works Under the Hood
1. The script recursively checks all `.png`, `.jpg`, and `.jpeg` files in the input folder.
2. It reads the metadata. If `width > max-dim`, `height > max-dim`, or `filesize > max-size-kb`, it loads the image array into RAM.
3. It scales the image down using Pillow's `Image.Resampling.LANCZOS`.
4. It checks for an `RGBA` or transparent `P` mode. If found, it creates a fresh white `RGB` canvas and pastes the image using its own alpha channel as a mask.
5. It saves the final array with deep optimization enabled (`optimize=True` for PNGs, `quality=85` for JPEGs).
